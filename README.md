#  Weather Dashboard

##  Краткое описание
Android-приложение для демонстрации работы Kotlin Coroutines в асинхронных операциях. Приложение имитирует загрузку погодных данных (температура, влажность, скорость ветра) из нескольких источников, показывая разницу между последовательной и параллельной загрузкой, работу с диспетчерами, обработку ошибок и автоматическое обновление UI без блокировки главного потока.

## 🛠 Функциональность приложения
-  Отображение трёх погодных параметров с индикаторами загрузки
-  Параллельная загрузка данных через `async`/`await` (~2 сек вместо ~4.5 сек)
-  Кнопка ручного обновления и автоматическое обновление каждые 10 секунд
-  Имитация сетевых ошибок и корректная обработка исключений через `coroutineScope`
-  Фоновый расчёт "Weather Index" на отдельном диспетчере вычислений
-  Современный реактивный интерфейс на Jetpack Compose с `StateFlow`
-  Адаптивная верстка и поддержка темной/светлой темы

##  Технологии и библиотеки
- **Язык:** Kotlin
- **UI:** Jetpack Compose, Material 3
- **Архитектура:** MVVM, `ViewModel`, `StateFlow` / `MutableStateFlow`
- **Асинхронность:** Kotlin Coroutines (`kotlinx-coroutines-android`, `kotlinx-coroutines-core`)
- **Жизненный цикл:** `androidx.lifecycle:lifecycle-viewmodel-compose`
- **Управление версиями:** Git + GitHub
- **Минимальная версия SDK:** API 25 (Android 7.1)

##  Ответы на контрольные вопросы

### A) В чём разница между `launch` и `async`?
- **Когда использовать:** 
  - `launch` используется для задач, результат которых не нужен (`fire-and-forget`), например, отправка логов или обновление состояния.
  - `async` используется, когда необходимо получить результат выполнения и дождаться его через `.await()`.
- **Пример кода:**
  ```kotlin
  // launch: запускает и забывает
  viewModelScope.launch { 
      sendLog("User refreshed weather") 
  }

  // async: запускает и возвращает результат
  val result = viewModelScope.async { calculateTemperatureIndex() }.await()

### B) Что такое suspend функция?
- Это функция, которая может приостанавливать и возобновлять своё выполнение без блокировки потока. Вызывается только из другой suspend функции или из корутины (launch/async).
- Может ли вызываться из обычной функции? Нет, компилятор Kotlin не позволит вызвать suspend функцию из обычного контекста.
- Почему delay() не блокирует поток? В отличие от Thread.sleep(), delay() приостанавливает только саму корутину, освобождая поток для выполнения других задач. Поток не блокируется, поэтому UI остаётся отзывчивым.

### C) Зачем нужны разные диспетчеры?
- Диспетчеры определяют, на каком потоке будет выполняться корутина. Это необходимо для разделения ответственности и предотвращения фризов интерфейса.
- Что будет, если выполнить тяжёлое вычисление на Dispatchers.Main? Главный поток будет заблокирован до завершения операции, интерфейс "зависнет", пользователь не сможет взаимодействовать с приложением, что может привести к ANR (Application Not Responding).

### D) Что произойдёт, если не обработать исключение в корутине?
- Если исключение возникнет в дочерней корутине async до вызова .await() и не будет перехвачено, оно всплывёт в родительскую корутину и вызовет FATAL EXCEPTION, что приведёт к аварийному закрытию приложения.
- Как корректно обрабатывать ошибки? Оборачивать запуск корутин или блок await() в конструкцию try-catch. Для группы независимых async задач рекомендуется использовать coroutineScope, который гарантирует, что ошибка одной задачи отменит остальные и безопасно пробросит исключение во внешний catch.
- Зачем нужен try-catch внутри launch? Чтобы предотвратить краш приложения, корректно обновить UI (например, показать сообщение об ошибке) и завершить загрузку в контролируемом режиме.
### E) Как работает автоматическая отмена корутин?
- Что такое viewModelScope? Это встроенная область корутин, привязанная к жизненному циклу ViewModel. Все корутины, запущенные внутри неё, автоматически наследуют её контекст.
- Когда корутины отменяются автоматически? При вызове ViewModel.onCleared(), что происходит, когда пользователь уходит с экрана или происходит пересоздание Activity, после которого ViewModel больше не нужна. viewModelScope отменяет все дочерние Job, предотвращая утечки памяти и фоновую работу после закрытия экрана.

  ## Как запустить проект
- Откройте проект в Android Studio (File → Open → выберите папку с проектом).
- Дождитесь автоматической синхронизации Gradle (или нажмите Sync Now).
- Нажмите  Run и выберите эмулятор или подключённое Android-устройство (минимум API 25).
